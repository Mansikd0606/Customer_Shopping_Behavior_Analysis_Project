# Customer_Shopping_Behavior_Analysis_Project

## 📌 Project Overview
The goal of this project is to simulate a corporate-grade end-to-end data analytics workflow, demonstrating the ability to translate raw data into strategic business intelligence by:

✅ Data Preparation,Modeling & Exploratory Data Analysis (Python): Clean and transform the raw dataset for analysis.

✅ Data Analysis (SQL): Simulate business transactions, and run queries to extract insights on customer segments, loyalty, and purchase drivers.

✅ Visualization & Insights (Power BI): Build an interactive dashboard that highlights key patterns and trends, enabling stakeholders to make data-driven decisions.

---

🎯 Objectives
 - Clean and preprocess raw customer data
 - Extract business insights using SQL queries
 - Build a visual dashboard for decision-making

---

🛠️ Tech Stack
 - Python – Data Cleaning & Preprocessing
 - Pandas – Data manipulation
 - SQL – Data extraction & analysis
 - Power BI – Data visualization

---

📂 Dataset
Dataset used: customer_shopping_behavior.csv

Contains customer-level transaction and behavioral data such as:
 - Customer demographics
 - Purchase history
 - Spending patterns

---

🔄 Project Workflow
1. Data Cleaning (Python)
 - Handled missing values
 - Removed duplicates
 - Standardized column formats
 - Converted data types

2. SQL Analysis
   
   Used SQL queries to extract key insights:
 - Top customers by spending
 - Revenue by category
 - Customer segmentation
 - Monthly/Yearly trends

3. Data Visualization
 - Built dashboard to visualize:
 - Total revenue
 - Customer distribution
 - Category-wise performance
 - Trend analysis

---

📊 Key Insights
 - Identified high-value customers contributing most to revenue
 - Found top-performing product categories
 - Observed purchasing trends over time
 - Discovered customer segmentation patterns
